Case study images live here.

Use one primary 16:9 hero image per case study, named by slug:

- `tkees-hero.jpg` or `tkees-hero.webp`
- `coppercow-hero.jpg` or `coppercow-hero.webp`
- `oftheancients-hero.jpg` or `oftheancients-hero.webp`
- `savenors-hero.jpg` or `savenors-hero.webp`

Preferred format: WebP for smaller files, but JPG is allowed.
Recommended size: 1600x900 or 1920x1080

When an image exists, set `heroImage` in the matching case study markdown file,
for example:

`heroImage: /work/tkees-hero.jpg`

This folder is source-controlled project input.
Do not place source images in `dist/`, which is Astro build output.
